How it works

The core functionality, the part of the lightbox that is so revolutionary, is the ability of CSS to recognize a click event. To do so is a hack, but uses valid css3 and html5. Css has been able to recognize a form field being clicked for a while with the :focus pseudo-class, but with HTML5's contenteditable attribute, any element can be "focused". When css recognizes the click it uses css3 transitions to animate the image.

There are some uncommon tricks that are used for user exprerience. To make the photos feel "real" every even numbered photo is rotated differently from odd numbered photos to give them almost a carefree appearance. Also, by registering the :active pseudo class in the same way as the :focus pseudo class, the animation begins , as a javascript even, onMouseDown instead of how most clicks are triggered, with the onMouseUp event.

Pros/Cons

Its Advantages

  • In the future, more people will be able to view css3 effects than will have javascript enabled
  • Fully supported by firefox 4.0+, safari 4.0+ and chrome (any version)
  • CSS3 transitions are much smoother and much less cpu intensive than javascript animations

Its Disadvantages

  • For now, many more people have javascript enabled than have css3 capable browsers
  • No support for IE... until a javascript fallback is made
